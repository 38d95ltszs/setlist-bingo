# セトリ予想ビンゴ

セットリスト予想ビンゴゲーム。複数の「大会」をプルダウンで選び、大会ごとに予想・ベット・確定・自動採点・ランキング・トレンドが楽しめる。

ビルド不要の素のHTML/CSS/JavaScript(単一ファイル)。GitHub Pagesでそのまま公開できる。バックエンドはSupabase(無料枠)。

## できること

- **予想タブ**: 5×5マスにセトリ予想、12ラインにベット、担当アイドル(アイドル名(C.V.声優名)形式)・ユーザー名・ひとことを入力
- **確定**: 締切前でも任意のタイミングで押せる(押すと予想内容がロックされる)。押さなくても締切時刻を迎えると自動的に確定扱いになる
- **自動採点(一括)**: 管理者が正解セトリを登録すると、確定済みの参加者**全員がその場で自動的に採点**される。各参加者がログインし直す必要はない
- **トレンドタブ**: 確定済み参加者全員の予想を集計した「みんなの予想傾向」(1曲目予想・ラスト曲予想・全体人気曲)。数時間おきに自動更新、管理者は手動でも即時集計できる
- **ランキングタブ**: 結果発表前は確定した順に、発表後は獲得ジュエルの多い順に一覧表示(ジュエル数・ガシャ換算・担当アイドル・ひとこと)
- **SNS共有**: セトリ予想ビンゴの図とベット状況を1枚のJPG画像にして生成し、保存またはXへ共有
- **管理タブ**: 大会作成者だけがアクセス可能。正解セトリのCSV登録、トレンド手動集計、締切変更、確定済み全員の再採点、大会削除ができる

## ライン別の払戻倍率(BINGO成立時)

| ライン | 倍率 | 理由 |
|---|---|---|
| 斜め↘ | ×10 | OPENING・ENCORE両方を含む(最難) |
| 横1・横5・縦1・縦5 | ×7 | OPENING/ENCOREのどちらか片方を含む |
| 横2・横4・縦2・縦4 | ×5 | 特別マスを含まない通常ライン |
| 横3・縦3・斜め↙ | ×3 | FREE(中央)を経由する(最も当てやすい) |

担当アイドルの声優(CV)が、BINGO成立ラインの曲の歌唱者と一致していると、該当マス1つにつきさらに×2。

## 候補曲・担当アイドルは全大会共通(静的ファイル)

`songs.json`(候補曲リスト)と `casts.json`(担当アイドル一覧、アイドル名+声優(CV))は、リポジトリ内の静的ファイルとして管理する。**全ての大会で共通**なので、大会ごとに登録する必要はない。差し替えたい場合は、このリポジトリ内の該当ファイルを直接編集してpushする。

```json
// songs.json の例
["モーニング・プリズム", "リハーサル・トワイライト", "..."]

// casts.json の例
[{"idol": "花海咲季", "cv": "石見舞菜香"}, {"idol": "月村手毬", "cv": "前田佳織里"}]
```

正解セトリ(実際に演奏された曲+歌唱者)は、これとは別に大会ごとにCSVで管理タブから登録する(下記参照)。

## 1. Supabaseプロジェクトを作る

1. https://supabase.com にログイン(なければアカウント作成)
2. 「New project」でプロジェクトを作成(リージョンは Tokyo (ap-northeast-1) を推奨)
3. 左メニュー **SQL Editor** を開き、`supabase-schema.sql` の中身を全部貼り付けて実行
   - `create extension if not exists pg_cron;` の行でエラーが出た場合(プランでpg_cronが使えない場合)は、その行と直後の `select cron.schedule(...)` をスキップして残りだけ実行してよい。その場合、トレンドは管理タブの「今すぐ集計する」ボタンでの手動運用になる
   - 既存プロジェクトに反映するだけの場合は、`alter table ... add column if not exists ...` や `drop policy if exists ... / create policy ...` の行だけ実行すれば、既存データを消さずに反映できる
4. 左メニュー **Authentication > Providers > Email** で **「Confirm email」をOFF**
5. 左メニュー **Project Settings > API** で **Project URL** と **anon public key** を確認

## 2. Supabaseの接続情報を書き込む

`index.html` 内、`<script type="module">` の先頭付近にある以下を書き換える。

```js
const SUPABASE_URL = "あなたのプロジェクトURL";
const SUPABASE_ANON_KEY = "あなたのanon key";
const SUPER_ADMIN_LOGIN_ID = "gakuen"; // 大会を作成できる管理者のログインID
```

> anon keyはpublicで問題ない値(Supabase設計上、フロントエンドに埋め込む前提のキー)。データの保護はRLS(Row Level Security)が担っている。

## 3. GitHubにアップロードする

```bash
cd gakumas-bingo
git init
git remote add origin https://github.com/38d95ltszs/gakumas-bingo.git
git add .
git commit -m "update"
git branch -M main

```

すでにpush済みのリポジトリなら、`git add . && git commit -m "update" && git push` でOK。

## 4. GitHub Pagesを有効化する

Settings > Pages で Source を `Deploy from a branch` / ブランチ `main` / フォルダ `/(root)` にする。

## 大会の使い方

1. `SUPER_ADMIN_LOGIN_ID` のIDでログインし、「＋ 新しい大会を作る」で大会名・開催日・締切を入力して作成(候補曲・担当アイドルは静的ファイルを共通で使うため入力不要)
2. 参加者は予想がまとまったら「確定する」を押す(締切前でもOK)。押さなければ締切時刻に自動確定
3. ライブ終了後、大会作成者は「管理」タブから正解セトリのCSVを登録する
   - CSV形式: 1行1曲、「曲名,歌唱者(声優/CV名)」。最初のカンマだけで区切るので、歌唱者が複数いる場合はそのままカンマ区切りでOK(例: `SUPREMACY,長月あおい, 小鹿なお, 飯田ヒカル`)
   - 「ユニット名(メンバー1, メンバー2)」形式や「全員」(登録済み声優全員として展開)にも対応
   - 登録すると、確定済みの参加者が自動で一括採点される
4. 「トレンド」タブでみんなの予想傾向を確認、「ランキング」タブで成績を確認できる

## ファイル構成

```
index.html           画面・見た目・ロジックすべてを含む単一ファイル
                      (★中盤のSupabase接続情報とSUPER_ADMIN_LOGIN_IDだけ書き換える)
songs.json            候補曲リスト(静的、全大会共通)
casts.json            担当アイドル一覧(静的、全大会共通。アイドル名+声優CV)
supabase-schema.sql   Supabaseに流し込むテーブル定義・RLS設定・トレンド集計関数
```

## 補足・今後の拡張余地

- 大会の管理者=作成者本人のみ(複数管理者には未対応)
- 一括採点・トレンド集計は、管理者の操作をきっかけに動く設計(数千〜数万人規模のアクセスを想定した完全サーバーレス分散処理ではなく、利便性を優先した設計)
- pg_cronがSupabaseのプランで使えない場合は、トレンド集計は管理タブの手動ボタンで運用する
